# 🚀 변방계 정착민 테스트 (RimWorld Colonist Test) 개발 일지

**프로젝트 개요**  
RimWorld(림월드)의 복잡하고 매력적인 캐릭터 생성 시스템을 웹 기반 심리 테스트로 구현한 프로젝트입니다. 사용자의 답변을 분석하여 림월드 스타일의 **특성(Traits)**, **기술(Skills)**, **결격 사항**, **배경 이야기**를 도출해냅니다.

---

## 🛠 기술 스택 (Tech Stack)

*   **Language**: TypeScript
*   **Styling**: Tailwind CSS
*   **State Management**: React Context API

## ✨ 핵심 기능 및 구현 과정 (Key Features)

### 1. 2단계 테스트 시스템 (Two-Phase Testing)
단순한 문답을 넘어, 캐릭터의 깊이를 더하기 위해 테스트를 두 단계로 분리했습니다.
*   **Phase 1 (성향 테스트):** 사용자의 기본적 성격, 도덕성, 생활 양식을 파악하여 **특성(Traits)**을 결정합니다.
*   **Phase 2 (적성 테스트):** 결과 페이지에서 '추가 검사'를 통해 진행되며, 구체적인 상황 대처 능력을 물어 **기술(Skills)**과 **열정(Passion)**을 측정합니다.
*   **구현:** `TestContext`에서 `testPhase` 상태('trait' | 'skill')를 관리하여 질문 데이터를 동적으로 필터링하고 UI를 전환합니다.

### 2. 스펙트럼 특성 시스템 (Spectrum Trait Logic)
단순히 점수가 높으면 특성을 주는 방식을 넘어, **상반된 특성을 하나의 축(Spectrum)으로 통합**했습니다.
*   **Logic:**
    *   예: **작업 속도 스펙트럼 (Work Speed)**
    *   점수 `+6` 이상: **근면성실 (Industrious)**
    *   점수 `+3` 이상: **일벌레 (Hard Worker)**
    *   점수 `-3` 이하: **게으름 (Lazy)**
    *   점수 `-6` 이하: **나태 (Slothful)**
*   이로써 사용자가 "부지런하다"와 "게으르다"를 동시에 선택하는 모순을 방지하고, 점수의 강도에 따라 더 강력한 특성이 발현되도록 설계했습니다.

### 3. 기술 열정 및 결격 (Passion & Incapability)
림월드의 꽃인 '불꽃(열정)'과 '결격' 시스템을 구현했습니다.
*   **쌍불(Major Passion) 🔥🔥:** 특정 질문에 대해 "이 일은 나의 천직이다" 수준의 대답을 하면 `+8`점의 보너스를 부여해 즉시 전설적인 성장 가능성을 부여합니다.
*   **결격(Incapable) 🚫:** "죽어도 하기 싫다"거나 "피를 무서워함" 등의 대답을 하면 해당 스킬에 `-20`점을 부여하여, **사실상 수행 불가능(레벨 0)** 상태로 만듭니다.

### 4. 고도화된 UI/UX 디자인
*   **ID 카드 컨셉:** 결과 화면을 실제 게임 내 캐릭터 정보창처럼 디자인하여 몰입감을 높였습니다.
*   **인터랙티브:**
    *   특성 클릭 시 상세 설명 패널 표시
    *   잠금된 스킬 슬롯이 2차 테스트 후 해금되는 애니메이션 효과
    *   모바일/데스크탑 반응형 레이아웃
*   **시각화:** 스킬 레벨을 Bar Graph로 시각화하고, 열정 여부를 불꽃 아이콘으로 직관적으로 표시했습니다.

### 5. 시스템 고도화 및 데이터 확장 (System Upgrades)
*   **결격 사항 UI 구현:**
    *   특정 테스트 질문(예: 전투 기피, 의학 공포)에 대한 답변 시 해당 스킬을 **'결격(Incapable)'** 처리.
    *   결과 화면에서 결격된 스킬은 레벨 막대 대신 붉은색 경고 태그로 표시하여 림월드 고증을 살림.
*   **랜덤 질문 샘플링 (Random Sampling):**
    *   매번 똑같은 질문이 나오는 것을 방지하기 위해 질문 풀(Pool)을 대폭 확장.
    *   **특성 질문:** 50+개 중 무작위 40개 추출.
    *   **스킬 질문:** 20+개 중 무작위 15개 추출.
*   **스킬 ID 1000번대 이전:** 확장성을 위해 스킬 관련 질문들의 ID 대역을 1000번대로 이동시켰습니다.

## 📂 데이터 구조 (Data Structure)

### `questions.json`
질문 데이터를 ID 기반으로 관리하며, 각 답변이 어떤 스탯에 영향을 주는지 매핑했습니다.
```json
{
  "id": 101,
  "text": "적의 습격! 당신의 손에는 저격소총이 들려있습니다.",
  "answers": [
    {
      "text": "호흡을 멈추고 적의 머리를 정확히 조준한다.",
      "scores": { "Shooting": 5, "careful_shooter": 2 }
    },
    {
      "text": "총은 너무 어렵다. 빗자루를 들고 돌격!",
      "scores": { "Shooting": -5, "Melee": 3 }
    }
  ]
}
```

### `traits.json`
약 40여 개의 림월드 특성을 정의하고, 그룹(Group)과 충돌(Conflicts) 정보를 포함했습니다.
```json
{
  "id": "beautiful",
  "name": "아름다움",
  "group": "외모",
  "effect": "관계도 +40",
  "conflicts": ["ugly", "staggeringly_ugly"]
}
```
