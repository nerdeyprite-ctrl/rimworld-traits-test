## 1. "나이 값"을 하는 정착민 (나이별 능력치 보정)

림월드에서 10대 소년병과 80대 노요원의 능력치가 같다면 말이 안 되겠죠. 기존의 단순 랜덤+가산점 방식을 버리고, **생물학적 연령에 따른 성장 곡선** 로직을 도입했습니다.

### 🛠️ 구현 로직
- **성장기 (< 20세)**: 아직 배우는 단계입니다. 능력치에 약간의 패널티(0.1 ~ 0.95배)를 부여하여 미숙함을 표현했습니다.
- **전성기 (20세 ~ 60세)**: 경험이 쌓이는 시기입니다. 나이가 들수록 능력치가 점진적으로 상승(1.0 ~ 1.5배)합니다.
- **노년기 (> 60세)**: 신체적 노화는 오겠지만, 기술적 숙련도는 정점에 달합니다. 경험치 보너스를 최대로 유지합니다.

```typescript
// Age Factor Logic Snippet
if (age < 20) {
    ageFactor = Math.max(0.1, age / 20); // 성장기
} else if (age <= 60) {
    ageFactor = 1.0 + ((age - 20) / 40) * 0.5; // 경험 축적
} else {
    ageFactor = 1.5; // 노련함
}
```

이로써 같은 선택지라도 20살 대학생과 50살 베테랑의 결과가 확연히 달라지는 재미를 주었습니다.

## 2. 특성(Trait) 스펙트럼 시스템

사람의 성격은 단면적이지 않습니다. 단순히 점수가 높다고 좋은 특성을 주는 게 아니라, **상반된 성향(Spectrum)** 개념을 적용했습니다.

### 🎯 스펙트럼 예시
*   **Mood (기분)**: 우울증 ↔ 낙천적
*   **Work (작업)**: 게으름 ↔ 부지런함 ↔ 직설적

사용자의 답변 점수가 양수(+) 방향으로 높으면 `직설적(Abrasive)`이나 `부지런함(Industrious)`을, 음수(-) 방향으로 낮으면 `다정다감(Kind)`이나 `게으름(Lazy)`이 부여되도록 설계했습니다. 이제 중도적인 답변을 피하고 확실한 성향을 보여야 강력한(혹은 치명적인) 특성을 얻게 됩니다.

## 3. UI/UX: 림월드 감성 한 스푼 더

기능만 좋으면 뭐하나요, 보기에 좋아야죠. 림월드 특유의 **'어둡지만 깔끔한(Dark & Minimal)'** 인터페이스 감성을 살려 UI를 다듬었습니다.

### 🎨 주요 변경점
1.  **배경 스토리 카드 통합**: 기존에 따로 놀던 `유년기`와 `성년기` 박스를 하나의 타임라인 컨테이너로 묶어, 캐릭터의 서사가 이어지는 느낌을 주었습니다.
2.  **색상 통일**: 중구난방이던 회색 톤을 정리하고, 핵심 포인트 컬러(림월드 오렌지 브라운)를 곳곳에 배치해 가독성을 높였습니다.
3.  **반응형 레이아웃**: 모바일에서도 쾌적하게 테스트를 진행할 수 있도록 버튼 크기와 여백을 재조정했습니다.
